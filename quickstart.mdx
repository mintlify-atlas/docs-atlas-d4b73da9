---
title: "Watchman quickstart: watch directories and subscribe"
description: "Watch a directory, run your first file query, and receive live change notifications with a CLI subscription or Node.js client in minutes."
---

This guide walks you through the core Watchman workflow: registering a directory watch, querying for files, and subscribing to real-time change notifications. By the end you will have a working subscription that prints file change events to your terminal.

<Note>
  Make sure Watchman is installed before continuing. See the [installation guide](/installation) for platform-specific instructions.
</Note>

<Steps>
  <Step title="Watch a directory">
    Use `watch-project` to register a directory with the Watchman service. This command is preferred over the lower-level `watch` command because it consolidates overlapping watches and respects project boundaries (detected via `.watchmanconfig` or version control root markers like `.git`).

    ```bash
    watchman watch-project ~/my-project
    ```

    Example output:

    ```json
    {
      "version": "2024.01.15.00",
      "watch": "/Users/you/my-project",
      "relative_path": null
    }
    ```

    <Note>
      If `relative_path` is present in the response, the watch was established at a parent directory. Any queries or subscriptions you create must use the `watch` path as the root, not the path you passed in. Use `relative_path` to scope your expressions to the subdirectory you care about.
    </Note>

    The Watchman service will crawl the directory tree, register OS-native change notifications with the kernel, and maintain a live in-memory index from this point forward.
  </Step>
  <Step title="Find files">
    Once a directory is watched, use `find` to list all files under a path:

    ```bash
    watchman find ~/my-project
    ```

    You can pass glob patterns to filter results:

    ```bash
    watchman find ~/my-project "*.ts" "*.tsx"
    ```

    `find` is useful for quick inspection but does not support the full expression language. For more powerful queries, use the `query` command in the next step.
  </Step>
  <Step title="Query for changes">
    The `query` command gives you full control over which files are returned. Queries accept a JSON expression object that filters files by type, name, size, modification time, and more.

    Use the `-j` flag to pass a JSON query from stdin:

    ```bash
    watchman -j <<-EOT
    ["query", "/Users/you/my-project", {
      "expression": ["allof",
        ["type", "f"],
        ["suffix", "ts"]
      ],
      "fields": ["name", "mtime", "size"]
    }]
    EOT
    ```

    Example output:

    ```json
    {
      "version": "2024.01.15.00",
      "clock": "c:1705312800:42",
      "is_fresh_instance": true,
      "files": [
        {
          "name": "src/index.ts",
          "mtime": 1705312750,
          "size": 2048
        },
        {
          "name": "src/utils.ts",
          "mtime": 1705312760,
          "size": 512
        }
      ]
    }
    ```

    To query only files that changed since a previous clock value, use the `since` generator:

    ```bash
    watchman -j <<-EOT
    ["query", "/Users/you/my-project", {
      "since": "c:1705312800:42",
      "expression": ["type", "f"],
      "fields": ["name", "exists", "new"]
    }]
    EOT
    ```

    <Note>
      Save the `clock` value from each query response. Passing it back as `since` in your next query returns only files that changed in the interval — this is far more efficient than re-querying all files.
    </Note>
  </Step>
  <Step title="Subscribe to live updates">
    Subscriptions push change notifications to your process as files change, without requiring you to poll. The connection stays open and the server sends results automatically after each filesystem settle period.

    ### CLI subscription

    Subscribe from the terminal with the `-p` (persistent) flag:

    ```bash
    watchman -j --server-encoding=json -p <<-EOT
    ["subscribe", "/Users/you/my-project", "my-watch", {
      "expression": ["allof",
        ["type", "f"],
        ["not", "empty"]
      ],
      "fields": ["name", "exists", "mtime"]
    }]
    EOT
    ```

    After the initial acknowledgement, the server sends a notification packet each time matching files change:

    ```json
    {
      "version": "2024.01.15.00",
      "subscribe": "my-watch"
    }
    ```

    ```json
    {
      "version": "2024.01.15.00",
      "clock": "c:1705312900:43",
      "root": "/Users/you/my-project",
      "subscription": "my-watch",
      "files": [
        {
          "name": "src/index.ts",
          "exists": true,
          "mtime": 1705312890
        }
      ]
    }
    ```

    ### Node.js subscription

    For programmatic use, the [`fb-watchman`](https://www.npmjs.com/package/fb-watchman) npm package wraps the Watchman socket protocol:

    ```bash
    npm install fb-watchman
    ```

    ```javascript watchman-subscribe.js
    const watchman = require('fb-watchman');
    const client = new watchman.Client();

    client.capabilityCheck(
      { optional: [], required: ['relative_root'] },
      (error, resp) => {
        if (error) {
          console.error('Capability check failed:', error);
          client.end();
          return;
        }

        const dir = process.cwd();

        client.command(['watch-project', dir], (error, resp) => {
          if (error) {
            console.error('watch-project failed:', error);
            client.end();
            return;
          }

          const watch = resp.watch;
          const relativeRoot = resp.relative_path;

          const sub = {
            expression: ['allof', ['type', 'f'], ['not', 'empty']],
            fields: ['name', 'exists', 'mtime'],
          };

          if (relativeRoot) {
            sub.relative_root = relativeRoot;
          }

          client.command(['subscribe', watch, 'my-sub', sub], (error) => {
            if (error) {
              console.error('subscribe failed:', error);
              client.end();
              return;
            }
            console.log('Watching', dir, 'for changes...');
          });
        });

        client.on('subscription', (resp) => {
          if (resp.subscription !== 'my-sub') return;
          console.log('Files changed:');
          resp.files.forEach((file) => {
            console.log(' ', file.name, file.exists ? '(modified)' : '(deleted)');
          });
        });
      }
    );
    ```

    Run the script and edit a file in your project directory to see notifications appear:

    ```bash
    node watchman-subscribe.js
    # Watching /Users/you/my-project for changes...
    # Files changed:
    #   src/index.ts (modified)
    ```

    <Note>
      The subscription is automatically removed when the client disconnects. To persist subscriptions across restarts, store the `clock` value from each notification and pass it as `since` when you re-subscribe, so you only receive changes that happened while your client was offline.
    </Note>
  </Step>
</Steps>
